# 未上市公司研究：多Agent並行深度研究框架

對 $ARGUMENTS 進行團隊化深度研究分析。專為螞蟻集團、小紅書、SpaceX、Stripe 等未上市公司設計。

**最終目標**：在資訊天然稀缺的條件下，儘可能還原這家公司的**真實價值**——不是市場給的估值，而是生意本身值多少錢。

## 框架特點

未上市公司 vs 上市公司研究的核心差異：
- **無標準化財報**：需多源拼湊、交叉驗證
- **估值錨定少**：依賴融資輪次、可比公司法、情景推演
- **資訊不對稱大**：需要更多"拼圖式"研究方法
- **退出路徑不確定**：IPO/併購/二級轉讓均有可能

## AI研究偏見自覺（本框架的核心前提）

未上市公司是AI研究偏見最嚴重的領域。必須時刻警惕以下陷阱：

**核心矛盾**：AI擅長結構化已有資訊，但未上市公司的資訊天然稀缺。這會導致：
1. **虛假保守**：因為資料少，AI傾向給出保守/模糊的結論，但資料少≠公司不好
2. **虛假精確**：為了填滿報告模板，AI可能用"合理推測"偽裝成"有據分析"
3. **對標陷阱**：強行與上市公司對標時，會繼承上市公司的估值邏輯，忽略未上市公司的獨特價值
4. **倖存者偏差**：網上能搜到的公司資訊往往有正面偏向（公司主動傳播的多是好訊息）

**應對原則**：
- 寧可留白說"不知道"，也不要用推測填滿表格偽裝確定性
- 每個資料點必須標註置信度（🟢高/🟡中/🔴低），讓讀者自己判斷
- 區分"可驗證的事實"與"AI的推理"，用不同格式標註
- 對於資訊極度稀缺的公司，切換為"第一性原理模式"——不追求報告完整，只回答幾個核心問題：
  1. 這門生意解決什麼真實問題？需求是真需求還是偽需求？
  2. 為什麼是這個團隊？他們有什麼獨特優勢？
  3. 如果成功了，天花板有多高？如果失敗了，最可能死在哪裡？
  4. 當前階段的關鍵驗證節點是什麼？

**反向利用資訊不對稱**：市場對未上市公司資訊少→定價效率低→恰恰可能是超額收益的來源。AI研究的目標不是消除資訊不對稱（做不到），而是在有限資訊中提取最關鍵的判斷依據。

## 財務資料拼湊（偵探式研究）

未上市公司沒有標準財報，需要多源拼湊、交叉驗證。每一個資料都要追溯到具體來源，標明時間和置信度。

### 2. 關鍵財務指標推算
儘可能推算以下資料，**每個資料點必須標明**：來源、時間、置信度、推算方法。

**收入端**：
- 總收入規模與增速（近3年，年度/季度顆粒度）
- 收入結構拆解（按業務線/產品線）
- 收入增長驅動因素分解：量（使用者/交易量）× 價（ARPU/客單價）
- 收入的季節性波動模式

**成本端**：
- 毛利率推算（對標同業上市公司，說明選取邏輯）
- 研發費用率推算（透過員工數 × 人均薪酬，或研發人員佔比推算）
- 銷售費用率推算（透過獲客渠道和投放資料推算）
- 管理費用率推算

**利潤端**：
- 經營利潤/EBITDA 推算
- 淨利潤/調整後淨利潤推算
- 盈利時間線：何時盈利？如果尚未盈利，預計何時？

**現金流端**：
- 經營現金流判斷（正/負，是否自我造血）
- 資本開支水平和趨勢
- 自由現金流推算
- 賬上現金/燒錢速度（透過融資金額、融資間隔、員工數推算）
- 現金跑道：按當前燒錢速度還能撐多久？

**效率指標**：
- 員工數量與人效（人均收入、人均利潤）
- 資本效率：每融1元錢產生多少收入
- 與同業上市公司效率指標對標

### 4. 融資歷史與估值演變
整理完整融資時間線：

| 輪次 | 時間 | 金額 | 投前估值 | 投後估值 | 領投方 | 跟投方 | 估值倍數增長 | 備註 |
|------|------|------|---------|---------|--------|--------|------------|------|

分析：
- 估值增長曲線是否健康（每輪增長倍數是否合理）
- 融資間隔是否合理（過於頻繁=燒錢快？過於稀疏=融資困難？）
- 是否出現估值倒掛（Down Round）
- 老股東是否持續加註（信心訊號）
- 最新一輪的投資條款推測：
  - 優先清算權（1x/2x/participating）
  - 反稀釋保護（full ratchet/weighted average）
  - 對賭條款（業績/上市時間）
  - 這些條款對普通股價值的影響

### 5. 估值分析（多方法交叉）

**方法1：最近融資估值法**
- 最近一輪估值及時間
- 考慮優先清算權等條款後的"普通股等價估值"（通常需要打折20-40%）
- 距今時間差異調整
- 這一輪投資人的動機分析（財務投資 vs 戰略投資，戰略投資者可能溢價）

**方法2：可比上市公司法**
- 選擇3-5家上市可比公司，說明選取理由
- 關鍵倍數對比：

| 可比公司 | P/S | P/E | EV/EBITDA | EV/Revenue | 增速 | 利潤率 |
|---------|-----|-----|-----------|------------|------|--------|

- 應用的調整：
  - 流動性折價：20-30%（標註具體取值和理由）
  - 增速溢價/折價
  - 規模折價
  - 監管/政策風險折價

**方法3：DCF 情景分析**
三種情景，每種都要列出關鍵假設：

| 假設 | 悲觀 | 中性 | 樂觀 |
|------|------|------|------|
| 未來5年收入CAGR | | | |
| 終態經營利潤率 | | | |
| 終態增長率 | | | |
| WACC | | | |
| 終值倍數(EV/EBITDA) | | | |

每個假設要有論據支撐，不能憑空假設。

**方法4：終局市值倒推法**
- 假設5年/10年後這門生意在終態時的市場地位
- 終態收入和利潤率假設
- 終態合理倍數（參考同行業成熟公司）
- 倒推當前合理估值區間
- 隱含的年化回報率

**方法5：交易對標法**
- 同行業近2年的併購/融資交易
- 交易倍數（P/S、P/E）
- 交易背景和溢價/折價因素

## 行業格局與競爭態勢

### 1. 行業定位與市場規模
- 公司所在的核心賽道定義（注意：公司自己定義的賽道可能是美化過的，要獨立判斷）
- TAM/SAM/SOM 三層市場規模：
  - TAM（總可定址市場）：整個大行業
  - SAM（可服務市場）：公司技術/模式能覆蓋的
  - SOM（可獲得市場）：當前實際能獲取的
- 市場規模資料來源對比（不同研究機構的預測可能差異很大）

## 重要注意事項

1. **6個Agent必須並行啟動**——在同一條訊息中呼叫6次Agent工具
2. **資料置信度標註**——未上市公司資料來源參差不齊，每個關鍵資料必須標註來源和置信度
3. **推算要透明**——所有推算過程要展示計算邏輯，不能憑空給數字
4. **交叉驗證**——關鍵資料至少2個來源交叉驗證，來源衝突時都列出
5. **訊號一致性檢驗**——彙總階段必須做跨維度的訊號一致性檢查
6. **結論要明確**——不迴避給出投資/觀望/迴避建議，但同時說明結論的置信度
7. **耐心等待**——6個Agent研究需要幾分鐘，實時向使用者更新進度
8. **中英文搜尋**——未上市公司資訊可能分佈在中英文媒體，需要兩種語言搜尋
9. **反偏見核心原則**——資料少≠公司不好，AI分析篇幅短≠投資確定性低。對於資訊極度稀缺的公司，切換"第一性原理模式"聚焦核心問題，不追求報告形式完整
10. **誠實留白**——報告中明確區分"有據分析"和"推測填充"，允許出現"此維度資料不足，無法給出有意義的結論"
11. **替代資料不是噪音**——招聘、專利、訴訟、App資料等替代資料可能比新聞報道更接近真實經營狀況
12. **真實價值導向**——最終目標是判斷這門生意值多少錢，不是輸出一份好看的報告。如果資訊不足以做出可靠的估值判斷，直接說"資訊不足，無法給出可靠估值"
