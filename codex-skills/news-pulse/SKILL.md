# 公司新聞脈搏：股價異動快速歸因團隊

對 $ARGUMENTS 進行最近新聞偵察與異動歸因。**這不是深度投研，是情報快速響應**——目標是 10 分鐘內回答："這家公司最近發生了什麼？股價異動的真因是什麼？要不要重審投資論文？"

## 適用場景

- 持倉/關注股票股價大漲/大跌（一般觸發線：單日 ±5%、一週 ±10%）
- 財報後股價異動，想快速搞清市場在反應什麼
- 看到新聞標題，但不確定是噪音還是真訊號
- **不適用**：完整投研（用 `/investment-team`）、財報深讀（用 `/earnings-review`）、長期論文跟蹤（用 `/thesis-tracker`）

## 執行流程

### 第一步：確認引數與場景

向使用者澄清以下資訊（如未在 $ARGUMENTS 中提供）：

| 引數 | 說明 | 預設 |
|------|------|------|
| **公司名** | 中文/英文/股票程式碼均可 | 必填 |
| **時間視窗** | 偵察新聞的回溯天數 | 預設 14 天，財報季可縮到 7 天 |
| **股價異動** | 漲/跌幅度 + 時間，如"跌 12%/3 天" | 選填，有則用於聚焦歸因 |
| **關注側重** | 公司事件 / 監管 / 行業 / 情緒 | 預設四方平均 |

如果使用者只說了公司名，先反問："最近多少天的新聞？有具體的股價異動要解釋嗎？"——**不要默默假設**。

### 第二步：資訊可得性分級

參考 `investment-team.md` 的 A/B/C 評級，但維度不同：

| 等級 | 特徵 | 偵察策略 |
|------|------|---------|
| **A 級（資訊充裕）** | 大盤股、媒體覆蓋廣、財報季 | 重點是**降噪和歸因**——資訊太多反而難找真因，每個 Agent 要有判斷力，過濾掉"重複轉述"的二手新聞 |
| **B 級（資訊適中）** | 中小盤、覆蓋一般 | 標準模式，每條關鍵事件附 1-2 個獨立信源 |
| **C 級（資訊稀缺）** | 港股小票、新上市、冷門 | 轉入"掃盲模式"——可能找不到任何能解釋異動的新聞，**這個結論本身就有價值**（可能是技術性/資金面而非基本面） |

將評級告知每個 Agent，影響其偵察方式。

### 第四步：建立 4 個偵察任務

使用 TaskCreate 建立以下 4 個任務：

#### 任務 1：公司事件偵察（company-event-scout）

- **subject**: `偵察 {公司名} 近 {N} 天的公司本體事件`
- **description**:
  1. **官方公告**：港交所/SEC/巨潮 等監管披露平台最近披露
  2. **財報與業績指引**：最新季報/年報、業績預告、業績會要點
  3. **管理層動作**：高管變動、增減持、回購、分紅、股權激勵
  4. **重大業務事件**：新產品釋出、併購重組、業務剝離、大客戶/大訂單
  5. **資本運作**：再融資、可轉債、ADR 轉換、回 A/退市動議
  6. **訴訟與合規**：被起訴、自行披露的合規事件
  7. 每條事件標註：**日期 / 來源連結 / 一句話摘要 / 與股價異動可能相關性（高/中/低）**
  8. 輸出按時間倒序的時間線表格

#### 任務 2：監管與政策（regulatory-watcher）

- **subject**: `偵察 {行業/公司} 近 {N} 天的監管與政策變化`
- **description**:
  1. **行業監管**：所在行業的新規、罰款、整改、牌照變化
  2. **跨境政策**：中美關係（中概股）、關稅、出口管制、資料安全
  3. **稅收政策**：增值稅、企業所得稅、個稅相關變化
  4. **反壟斷與競爭法**：調查、罰款、併購否決
  5. **特殊行業政策**：醫藥集採、教育雙減、地產三道紅線、網際網路平台規制等
  6. **貨幣與外匯**：影響該公司的匯率/利率/資本管制變化
  7. 每條政策標註：**日期 / 來源 / 直接影響該公司的程度（直接/間接/無關）**
  8. 重點判斷：是否有"政策黑天鵝"剛剛落地

#### 任務 3：行業與競爭對手（industry-peer-analyst）

- **subject**: `偵察 {公司名} 行業格局與對手近 {N} 天的動態`
- **description**:
  1. **直接對手**：列出 3-5 個核心競爭對手，逐個查最近事件（財報、產品、價格戰、人事）
  2. **產業鏈上下游**：上游原材料/供應商、下游客戶/渠道，最近的價格、產能、訂單變化
  3. **行業整體**：行業景氣度資料、出貨量、需求側訊號（消費資料、招標資料等）
  4. **替代品威脅**：新技術、新業態對該行業的衝擊
  5. **行業指數表現**：同行業股票最近表現，該公司是跑贏/跑輸/同步
  6. 關鍵判斷：**這是公司個體事件，還是整個行業的 beta 波動？**
  7. 每條事件標註來源和日期

#### 任務 4：市場情緒與賣方/大V（sentiment-tracker）

- **subject**: `偵察 {公司名} 近 {N} 天的市場情緒與機構觀點變化`
- **description**:
  1. **賣方評級變動**：高盛、摩根、中金等最近的評級/目標價調整
  2. **機構持倉變化**：13F 披露（美股）、港股通持倉、北上資金流向
  3. **做空資料**：做空比例、新發布的做空報告（如有）
  4. **大 V 觀點**：可呼叫 `python3 tools/xueqiu_scraper.py` 抓段永平等大 V 最近相關發言
     - 段永平 user_id: `1247347556`
     - 命令示例：`python3 tools/xueqiu_scraper.py --user-id 1247347556 --keywords {公司名},{股票程式碼} --output /tmp/dyp-{公司名}.md`
     - 僅在該公司是段永平/李錄關注標的時呼叫，否則跳過節省時間
  5. **傳言與小作文**：媒體未證實的傳言、社交媒體討論熱點（雪球/X/Reddit）
  6. **技術面訊號**：是否觸及關鍵支撐/阻力、是否有大宗交易、融資融券異常
  7. 關鍵判斷：**是基本面驅動還是情緒/資金面驅動？**

### 第七步：team-lead 綜合歸因

彙總 4 份偵察報告，輸出**異動歸因報告**（不是研究報告，重點是"判斷"）：

---

#### 1. 一句話歸因
> 用一段話（30-60 字）說明：這次股價異動的主因 + 次因 + 性質判斷（價值事件/情緒波動/不明）

#### 2. 完整事件時間線（合併 4 個維度）

按日期倒序，合併所有維度的事件：

| 日期 | 維度 | 事件 | 來源 | 異動歸因權重 |
|------|------|------|------|-----------|
| 2026-04-30 | 公司 | XX | 連結 | 🔴 高 |
| 2026-04-29 | 行業 | XX | 連結 | 🟡 中 |
| 2026-04-28 | 情緒 | XX | 連結 | ⚪ 低 |

權重圖例：🔴 高（足以單獨解釋異動）/ 🟡 中（貢獻一部分）/ ⚪ 低（背景噪音）

#### 3. 異動歸因表

| 候選解釋 | 證據 | 反證 | 置信度 | 持續性 |
|---------|------|------|------|--------|
| 例：財報 miss | 營收低於預期 5%、毛利率下滑 | 一次性因素管理層有解釋 | 高 | 短期 1-2 周 |
| 例：行業 beta | 同行同期跌 8% | 該股跌幅明顯大於行業 | 中 | 與行業同步 |

#### 4. 性質判斷（核心結論）

打勾其一：

- [ ] **價值事件**：基本面發生了真實變化（業績、護城河、管理層、終局），需要重審投資論文
- [ ] **情緒/技術波動**：基本面無變化，是資金面/情緒/Beta 驅動，可視為機會或噪音
- [ ] **真因不明**：找不到能匹配股價異動幅度的事件——**這是最危險的結論**，要麼是市場提前知道了什麼（內幕/搶跑），要麼是我們漏掉了資訊源
- [ ] **混合**：部分價值事件 + 部分情緒放大

#### 5. 各維度偵察摘要

每維度 3-5 條最重要發現 + 該維度歸因貢獻度。

#### 6. 行動建議

| 行動 | 是否建議 | 理由 |
|------|--------|------|
| 觸發投資論文重審（`/thesis-tracker`） | | |
| 觸發深度財報研讀（`/earnings-review`） | | |
| 觸發管理層重審（`/management-deep-dive`） | | |
| 調倉動作（加倉/減倉/不動） | | 僅做提示，最終決策權在使用者 |
| 僅觀察 | | |

#### 7. 接下來 7-30 天的跟蹤清單

- [ ] 待披露事件 1（如：5/15 業績會）
- [ ] 待跟蹤指標 2
- [ ] 關鍵觀察訊號 3

## 關鍵原則

1. **快勝過全**——這個 skill 的核心價值是 10-15 分鐘內給出歸因判斷，不要陷入深度分析（那是其他 skill 的職責）
2. **歸因優先於羅列**——發現事件不難，難的是判斷"哪個事件配得上這次股價異動"。要做減法，不要做加法
3. **誠實面對"不明"**——找不到主因時，明確寫"真因不明"。這比硬湊因果鏈更有價值（市場可能在搶跑利空訊息）
4. **不預設立場**——不要因為持倉就傾向於"這是情緒波動，沒事"。證據指向哪邊就寫哪邊
5. **區分"催化劑"和"巧合"**——同一時間發生的事件不一定是異動主因，要看影響量級是否匹配
6. **尊重資訊可得性**——C 級公司可能就是查不到任何新聞，這個結論本身就要寫出來
7. **遵循 `CLAUDE.md` 客觀性原則**——所有判斷附資料來源，區分事實與觀點
8. **不替使用者做決策**——給出歸因和行動建議清單，但買賣決策由使用者做
