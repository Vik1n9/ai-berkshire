# 巴菲特價值投資買入前 Checklist

對 $ARGUMENTS 執行巴菲特價值投資買入前 Checklist 分析。

**支援輸入格式**：單個或多個公司，用逗號/頓號/空格分隔。例如：`騰訊, 茅台, 英偉達` 或 `NVDA AAPL MSFT`

## 執行流程

### 第一步：解析輸入，識別所有待分析公司

從 $ARGUMENTS 中解析出所有公司名稱/程式碼。對每家公司確定：
- 公司全稱、股票程式碼、上市交易所
- 如果公司未上市，標記為"未上市"並給出簡要說明（是否有間接投資途徑），跳過完整Checklist

### 第一步半：AI研究偏見預警

對每家公司進行"資訊豐富度"快速評級（A/B/C），並在報告中標註：

| 等級 | 判斷標準 | 對Checklist的影響 |
|------|---------|-----------------|
| A級 | 上市多年、資料充裕 | 正常執行，但警惕"共識陷阱"——所有指標看起來都清晰不代表真的確定 |
| B級 | 資料有限需推算 | 每個推算指標標註置信度，"好生意"判斷加權考慮資料可靠性 |
| C級 | 資訊極度稀缺 | 不勉強填滿六關表格，誠實標註"資料不足無法判斷"，聚焦可驗證的核心問題 |

**核心原則**：Checklist的目標是**排除壞選擇**。對於C級公司，"資料不足"不等於"不透過"，也不等於"透過"——應誠實標註為"灰色地帶，需補充一手資訊"，而不是因為AI無法填滿表格就判為否決。

段永平說過："看不懂"有兩種——一種是生意太複雜真的看不懂，一種是你還沒花時間去看。AI研究的侷限是容易把"資料少"和"看不懂"混為一談。

### 第二步：並行資料收集

使用 Task 工具為**每家公司**啟動獨立的後台 Agent 進行資料收集（所有公司同時並行啟動），每個Agent負責收集：

1. **盈利能力**：ROE（5-10年趨勢）、毛利率、淨利率、自由現金流
2. **估值資料**：當前股價、市值、PE(TTM)、前瞻PE、PB、股息率
3. **增長趨勢**：近3年收入/利潤增速
4. **財務健康**：負債水平、資本開支需求、現金儲備、淨現金/淨負債
5. **競爭格局**：市場份額、主要競爭對手、份額變化趨勢
6. **護城河證據**：品牌/轉換成本/網路效應/規模效應/技術壁壘的具體證據
7. **管理層記錄**：CEO履歷、關鍵決策、持股、資本配置記錄
8. **最新動態**：近6個月重大事件（業績、併購、監管、管理層變動等）

### 第三步：逐公司執行六關 Checklist

對每家已上市公司，依次過六關：

---

#### 第一關：我能理解這門生意嗎（能力圈）

必須回答：
- [ ] 能用一句話說清楚這家公司靠什麼賺錢嗎？
- [ ] 10年後大機率還在做什麼生意？
- [ ] 哪些關鍵變數決定成敗？
- [ ] 對這個行業的認知是來自深度研究還是道聽途說？

**評分標準**（★1-5）：
- ★★★★★：商業模式極其簡單清晰，10年確定性高（如茅台：釀酒賣酒）
- ★★★★☆：模式清晰但有技術門檻，需要一定專業知識理解
- ★★★☆☆：模式可理解但10年確定性不高，行業變化快
- ★★☆☆☆：業務線複雜或行業劇變中，難以預判未來
- ★☆☆☆☆：完全不在能力圈內

**硬性否決**：如果連賺錢方式都說不清，直接標記為"不在能力圈，不做分析"。

| 指標 | 該公司數值 | 參考標準 | 判斷 |
|------|-----------|---------|------|
| ROE（5年均值） | | >15%優秀, >20%卓越 | |
| 毛利率 | | >40%暗示定價權 | |
| 自由現金流 | | 持續為正、≈淨利潤 | |
| 資本開支強度 | | 輕資產優於重資產 | |
| 負債水平 | | 有息負債/淨利潤<3年 | |

**評分標準**（★1-5）：

**5句話說不完整 = 不買。** 明確標註"透過"或"未透過"。

## 輸出格式要求

1. 每家公司獨立成章，包含：六關評分表 + 核心資料表 + 關鍵風險（3-5條）+ 鏡子測試 + 明確結論
2. 多公司時在最後附總覽對比表
3. 所有評分必須使用★符號（★1-5），不含半星
4. 資料必須標註來源時間，估計值必須註明"估計"
5. 文末附結語，呼應巴菲特名言："投資的第一條規則是不要虧損"
6. 語言風格：直接、犀利、不說廢話。用巴菲特/芒格/段永平的語錄穿插點評

## 關鍵原則

- **寧可錯過，不可做錯**：Checklist的目標是排除壞選擇，不是找到最好的
- **誠實面對能力圈**：看不懂就說看不懂，不要勉強分析
- **安全邊際是生命線**：好公司買貴了也會虧錢
- **鏡子測試不可跳過**：說不清楚理由就不買，沒有例外
