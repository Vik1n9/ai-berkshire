# 供應鏈瓶頸獵手：AI驅動的全球產業鏈瓶頸套利

對 $ARGUMENTS 超級趨勢執行供應鏈瓶頸掃描與套利機會挖掘。

## 核心理念

不問"AI推薦什麼股票"，問"如果這個趨勢繼續擴張，哪一環會先不夠用？"

傳統投研盯龍頭、盯已知賽道。這個系統反過來：**從物理供應鏈的咽喉位置出發，找那些沒人注意但一旦斷貨整個行業都得停下來等的公司**。

超額收益來源：第一層瓶頸（GPU、HBM、電力）已被充分定價。真正的alpha在**第二層、第三層**——光模組、鐳射器、InP襯底、SOI晶圓、外延裝置、晶圓級測試、IC載板、特殊玻纖等。

## 第一步：超級趨勢確認

### 1.1 趨勢篩選標準

不在小風口裡找幻覺，只追符合以下全部條件的超級趨勢：

| 標準 | 要求 | 驗證方法 |
|------|------|---------|
| 持續性 | 至少3-5年確定性增長 | 搜尋行業預測、資本開支計劃 |
| 物理性 | 需要實際硬體/材料/裝置建設 | 區分"軟體升級"和"物理擴建" |

| 加速性 | 需求增速>供給擴產速度 | 對比需求增長率vs產能擴張計劃 |

## 第二步：供應鏈物理拆解

### 2.1 分層拆解框架

**不停留在概念層，拆到物理實體**。

```
Layer 0（終端）：最終產品/服務
    │
Layer 1（核心元件）：已被市場充分關注的核心硬體
    │                 ⬆ 定價充分，alpha有限
    │─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
    │                 ⬇ 關注度低，alpha集中區
    │
Layer 2（子元件/材料）：支撐核心元件的零部件和材料
    │
Layer 3（上游裝置/原料）：製造子元件所需的裝置和原材料
    │
Layer 4（基礎設施）：電力、冷卻、土地、人才、認證
```

### 3.1 瓶頸判定6條標準

## 第四步：公司篩選——從瓶頸到標的

### 4.1 對每個S級和A級瓶頸，找出所有相關上市公司

搜尋方式：
- WebSearch "{瓶頸環節} supplier listed company"
- WebSearch "{瓶頸環節} manufacturer stock"
- WebSearch "{瓶頸產品} market share company"

### 4.2 初篩標準（快速過濾）

| 標準 | 要求 | 理由 |
|------|------|------|
| 上市狀態 | 已上市（A/港/美/日/台/歐） | 可交易 |
| 瓶頸業務佔比 | >30%收入來自瓶頸環節 | 純正度 |
| 市值 | 優先<100億美元 | 大市值已被充分定價 |
| 流動性 | 日均成交額>100萬美元 | 可進出 |

### 4.2.1 估值檢查（必須執行，不可跳過）

**瓶頸真實 ≠ 投資機會。** 必須對每家公司計算 PS、PE 並標註在報告中。以下用組合條件判斷估值是否透支：

#### 估值紅燈（滿足任意一條 → 訊號強度封頂 ★★，標註"⚠️ 估值透支"）

1. **市值 > TAM的20%**：公司市值已超過其可觸達市場規模的20%，說明增長預期已過度內化
2. **PS > 30x 且 收入增速 < 100%**：高估值但增速不足以支撐。增速>100%的公司可豁免PS紅線，但仍需標註"⚠️ 高估值需高增速持續驗證"
3. **市值 > 5年樂觀收入預測的10倍**：即使最樂觀假設全部兌現，當前定價仍然過高
4. **增發後60天內股價翻倍**：情緒驅動特徵明顯，訊號強度降一級

#### 估值黃燈（需要額外解釋，否則降級）

1. **虧損 + PS > 15x**：允許進入 ★★★ 但必須說明盈利路徑和時間表
2. **PS 是同行盈利公司的 5 倍以上**：必須解釋溢價來源（市場份額、增速差、壁壘差異）
3. **PE > 80x**：需要計算 PEG 並說明增速是否支撐

#### 估值綠燈（加分項）

- PS < 10x 且收入在增長 → 訊號強度可加一級
- PE < 30x 且有護城河 → 標註"估值具有安全邊際"

#### 估值合理性檢驗（必做）

對每家標的回答："以當前市值買入，假設最樂觀情景全部兌現，10年後以25x PE退出，年化回報是多少？" 年化回報<10%→標註"當前價格不具備安全邊際"。

**注意**：估值檢查的目的是防止推薦"PS 100倍的虧損公司"這種明顯錯誤，而不是排除所有高估值早期公司。關鍵是增速、TAM、競爭格局能否支撐當前估值，需要具體分析而非一刀切。

**台股標的取數**：供應鏈掃描常命中台股（4位數字程式碼，如 2330、3661），其行情/PER/市值/月營收統一用 `python3 tools/twstock_data.py`（FinMind 資料來源，用法見 `skills/financial-data.md` 台股章節）。台股每月10日前強制披露上月營收，月營收同比是驗證瓶頸"量價齊升"最快的公開訊號。

## 第五步：交叉驗證——不只聽一個故事

### 5.1 正向驗證

| 驗證項 | 問題 | 搜尋方式 |
|--------|------|---------|
| 客戶驗證 | 頭部客戶是否已簽約/匯入？ | 搜尋公司公告、客戶財報提及 |
| 收入驗證 | 瓶頸是否已體現在收入增長中？ | 搜尋最近2-3個季度財報 |
| 價格驗證 | 產品是否在漲價？ | 搜尋行業報價、分析師報告 |
| 產能驗證 | 產能是否真的緊張？ | 搜尋交期資料、客戶抱怨 |
| 資本驗證 | 是否有擴產資本開支？ | 搜尋公司capex指引 |

### 5.2 反向驗證（芒格式否定）

| 反向問題 | 意義 |
|---------|------|
| 聰明人為什麼不買這隻股票？ | 尋找已知的bearish論點 |
| 這個瓶頸能被繞過嗎？有什麼替代路線？ | 技術路線風險 |
| 中國/其他玩家能不能很快複製產能？ | 供給衝擊風險 |
| 終端需求如果放緩50%，這家公司會怎樣？ | 下行敏感度 |
| 管理層過去有沒有在高點增發稀釋？ | 管理層信任度 |
| 當前估值隱含了什麼增長假設？ | 估值合理性 |

### 5.3 訊號交叉驗證

- 同一瓶頸的多家公司是否都在漲？（行業驗證）
- 下游客戶是否在財報中提到供應緊張？（客戶驗證）
- 行業協會/研究機構是否有相關資料？（第三方驗證）

## 每小時掃描模式（定時任務使用）

每小時執行一次，採用"只在有料時出報告"的模式：

### 掃描流程（每小時）

1. **新聞掃描**：搜尋過去1-2小時的供應鏈相關新聞
   - 關鍵詞：supply chain bottleneck, shortage, capacity constraint, allocation, lead time, sole source, 瓶頸, 缺貨, 產能, 漲價
   - 覆蓋：英文+中文源
2. **市場訊號**：檢查已跟蹤公司的股價變化（特別關注異常波動>5%）
3. **財報/公告**：檢查是否有瓶頸相關公司釋出財報或重大公告
4. **估值機會**：檢查watchlist中公司是否有因大盤下跌等原因進入買入區間
5. **判斷是否出報告**：
   - 有新瓶頸訊號、有明確標的機會、有重大狀態變化 → **出報告**
   - 無新發現 → **不出報告**，僅在日誌中記錄"本輪無新訊號"

| 偏見 | 表現 | 應對 |
|------|-----|------|

| 英文偏好 | 遺漏日韓台公司 | 必須搜尋日/韓/台灣市場的供應商 |
| 敘事偏好 | 被"AI概念"標籤吸引 | 只看實際供應鏈位置，不看市場標籤 |
| 確認偏見 | 找到瓶頸後只找正面證據 | 強制執行反向驗證（第五步） |
| 時效偏見 | 依賴過時資訊 | 優先搜尋最近30天的資料 |

## 核心原則（最高優先順序）

1. **不讓AI推薦股票，讓AI拆供應鏈** — 問題比答案重要
2. **物理優先** — 只關注需要實際物理產品/材料/裝置的環節
3. **第二層、第三層** — 不追已被充分定價的龍頭
4. **交叉驗證** — 每個結論至少2個獨立信源
5. **誠實面對不確定性** — 找不到資料就寫"資料不足"，不用推測填充
6. **瓶頸有時效性** — 每個瓶頸都會被解除，關鍵是判斷時間視窗
7. **小市值≠好機會** — 小市值也可能是爛公司，必須過財務質量關
8. **瓶頸真實≠投資機會** — 一家公司可以坐在最緊的瓶頸上，但如果PS>30x或仍在虧損，當前價格就不是買點。**估值是硬門檻，不可被瓶頸純正度、訊號強度或敘事吸引力覆蓋。** 寧可錯過一個漲了的瓶頸股，也不要在100倍PS買入一家虧損公司
9. **遵循CLAUDE.md客觀性原則** — 不預設看多，先資料後結論
