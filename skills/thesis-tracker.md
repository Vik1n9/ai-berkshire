# 投資論文追蹤：買入後的紀律系統

對 $ARGUMENTS 執行投資論文追蹤檢查。

**支援輸入格式**：
- `公司名` — 首次使用時建立投資論文，後續使用時追蹤檢查
- `公司名 建立論文` — 強制重新建立投資論文
- `公司名 季度檢查` — 基於最新財報進行論文檢查

> "買入只是開始。真正的工作是持有期間的持續跟蹤。" —— 李錄
>
> "當事實改變時，我就改變想法。你呢？" —— 凱恩斯

## 設計理念

大多數投資者的流程是：研究 → 買入 → 祈禱。缺少買入後的系統化跟蹤，導致：
- 該賣的時候捨不得賣（"再等等，會漲回來的"）
- 不該賣的時候恐慌賣出（"跌了20%，是不是我錯了"）
- 忘記了當初為什麼買的（"我買這個是因為什麼來著？"）

巴菲特和李錄的做法是：**買入前就寫下賣出條件**。然後每個季度檢查論文是否完整。

## 執行流程

### 第一步：判斷操作模式

檢查是否已存在該公司的投資論文檔案（`reports/{公司名}-thesis.md`）：
- 如果不存在 → 進入**建立論文**模式
- 如果存在 → 進入**追蹤檢查**模式
- 如果找不到但使用者表示已有 → 詢問檔案路徑

## 模式A：建立投資論文

### A0：資料收集

使用 WebSearch 獲取當前股價、估值指標（PE/PB/股息率）、最新財報核心資料，用於填寫估值錨點。如果已有該公司的 `/investment-research` 或 `/investment-team` 報告，優先從中讀取。

使用 `tools/financial_rigor.py verify-valuation` 校驗估值資料。

### A1：核心論文（必須用200字以內寫清楚）

投資論文必須回答以下5個問題，每個問題一句話：

```
我以 ___元 買入 ___公司，因為：
1. 這門生意的本質是___，我理解它的賺錢方式
2. 它的護城河是___，而且在變寬/穩定
3. 管理層___，值得信賴的原因是___
4. 當前價格相當於內在價值的___折，安全邊際來自___
5. 即使我錯了，下行風險可控，因為___
```

**如果5句話寫不完整，這個論文字身就有問題——說明買入決策不夠清晰。**

### A2：核心假設清單

把投資論文拆解成可驗證的具體假設：

| # | 核心假設 | 驗證方式 | 驗證頻率 | 當前狀態 |
|---|---------|---------|---------|---------|
| 1 | 例：收入增速維持15%+ | 季報收入增速 | 每季度 | 🟢 成立 |
| 2 | 例：毛利率穩定在60%+ | 季報毛利率 | 每季度 | 🟢 成立 |
| 3 | 例：管理層持續回購 | 回購公告/現金流表 | 每季度 | 🟢 成立 |
| 4 | 例：競爭對手未取得突破 | 行業資料/競對財報 | 每半年 | 🟢 成立 |
| 5 | ... | ... | ... | ... |

通常3-7個假設。太少說明思考不夠深入，太多說明論文不夠聚焦。

### A3：紅線清單（觸發任何一條 = 必須重新評估）

| # | 紅線條件 | 嚴重程度 | 觸發後動作 |
|---|---------|---------|-----------|
| 1 | 例：管理層誠信出問題（財務造假、關聯交易） | 致命 | 立即清倉 |
| 2 | 例：核心業務連續2季度收入下滑 | 嚴重 | 減倉50%，重新評估 |
| 3 | 例：護城河被明確突破（競對獲得同等能力） | 嚴重 | 啟動深度研究，考慮退出 |
| 4 | 例：監管政策根本性改變商業模式 | 嚴重 | 重新評估內在價值 |
| 5 | 例：管理層大規模減持（非計劃性） | 警告 | 深入調查原因 |

**段永平**："賣出只有三個理由：1.發現買錯了；2.公司基本面變了；3.找到了更好的。"

### A4：估值錨點

| 指標 | 買入時 | 樂觀目標 | 中性目標 | 悲觀情景 |
|------|-------|---------|---------|---------|
| 股價 | | | | |
| PE | | | | |
| 市值 | | | | |
| 內在價值估算 | | | | |
| 安全邊際 | | | | |

### A5：儲存論文

將投資論文寫入 `reports/{公司名}-thesis.md`，包含：
- 建立日期
- 買入價格和倉位
- 核心論文（5句話）
- 核心假設清單
- 紅線清單
- 估值錨點
- 追蹤記錄表（初始為空）

## 模式B：追蹤檢查

### B1：讀取現有論文

讀取 `reports/{公司名}-thesis.md`，載入：
- 核心論文
- 核心假設清單
- 紅線清單
- 上次檢查記錄

### B2：收集最新資料

使用 WebSearch 收集：
1. 最新財報資料（如果有新的季報/年報）
2. 近期重大事件（管理層變動、監管政策、競爭動態）
3. 當前股價和估值指標
4. 內部人交易記錄（大股東增減持）

### B3：逐條檢查核心假設

對每個核心假設，用最新資料驗證：

| # | 核心假設 | 上次狀態 | 最新證據 | 當前狀態 | 變化 |
|---|---------|---------|---------|---------|------|
| 1 | 收入增速15%+ | 🟢 成立 | Q4收入增速12% | 🟡 邊際弱化 | ⚠️ |
| 2 | 毛利率60%+ | 🟢 成立 | 毛利率61.2% | 🟢 成立 | — |
| 3 | ... | ... | ... | ... | ... |

狀態定義：
- 🟢 **成立** — 最新資料支援該假設
- 🟡 **邊際弱化** — 資料仍在可接受範圍，但趨勢不利
- 🔴 **受損** — 資料明確不支援該假設
- ⚫ **破裂** — 假設已被推翻

### B4：紅線檢查

逐條檢查紅線清單：

| # | 紅線條件 | 是否觸發 | 證據 |
|---|---------|:-------:|------|
| 1 | 管理層誠信問題 | ❌ 未觸發 | — |
| 2 | 核心業務連續2季下滑 | ❌ 未觸發 | — |

**任何一條紅線觸發 → 在報告中用醒目標註，給出明確的行動建議。**

### B5：估值更新

| 指標 | 買入時 | 上次檢查 | 當前 | 變化 |
|------|-------|---------|------|------|
| 股價 | | | | |
| PE(TTM) | | | | |
| 內在價值估算 | | | | |
| 安全邊際 | | | | |

### B6：輸出追蹤報告

#### 報告結構

```
一、論文健康度評分（滿分10分）
二、核心假設檢查結果（表格）
三、紅線檢查結果（表格）
四、本期關鍵變化（不超過500字）
五、估值更新
六、結論與行動建議
七、下次檢查需關注的重點
```

#### 論文健康度評分標準

| 評分 | 含義 | 建議動作 |
|:----:|------|---------|
**計算公式**：健康度 = 10 - (⚫破裂假設數×3) - (🔴受損假設數×2) - (🟡弱化假設數×1) - (紅線觸發數×5)，最低1分最高10分。

| 評分 | 含義 | 建議動作 |
|:----:|------|---------|
| 9-10 | 所有假設成立，論文比買入時更強 | 考慮加倉 |
| 7-8 | 核心假設成立，個別邊際弱化 | 繼續持有 |
| 5-6 | 1-2個假設受損，但核心邏輯未變 | 持有但提高警惕 |
| 3-4 | 多個假設受損，論文基礎動搖 | 考慮減倉 |
| 1-2 | 紅線觸發或核心假設破裂 | 強烈建議賣出 |

#### 結論必須明確回答

1. **論文還完整嗎？** 完整 / 邊際弱化 / 受損 / 破裂
2. **該怎麼做？** 加倉 / 持有 / 減倉 / 清倉
3. **下次檢查時間**：下一個季報釋出後 / 某個特定事件後

### B7：更新論文檔案

將本次檢查記錄追加到 `reports/{公司名}-thesis.md` 的追蹤記錄表中：

| 檢查日期 | 健康度 | 核心變化 | 動作建議 |
|---------|:------:|---------|---------|
| 2026-04-09 | 7/10 | 收入增速放緩至12%，但利潤率改善 | 持有 |

## 關鍵原則

- **買入前就寫好賣出條件** — 冷靜時做的決策比恐慌時做的好
- **論文要具體到可驗證** — "公司很好"不是論文，"ROE>25%且趨勢穩定"才是
- **紅線一旦觸發就行動** — 最怕的是"再等等看"，這是虧大錢的開始
- **論文破裂 ≠ 股價下跌** — 股價跌30%不一定要賣，論文破裂才要賣
- **誠實面對錯誤** — 論文建錯了就承認，不要為了面子硬撐
