# 行業漏斗篩選：從全市場到 3 家的價值投資精選流程

對 $ARGUMENTS 行業/方向執行漏斗式價值投資篩選，從全市場掃描逐層精選到 3 家終選標的。

## 適用場景

當你說出一個行業或投資方向（如"AI 算力"、"創新藥"、"機器人"），想要：
1. 不遺漏任何重要標的（含 A 股、港股、美股、未上市候選）
2. 用統一標準過濾掉"故事股"和質量不足的公司
3. 把精力聚焦到真正值得深度研究的 3 家頭部
4. 每層有明確的留/棄標準，可覆盤可追溯

與 `industry-research` 的區別：
- `industry-research` 偏重產業鏈結構與全景，環節切片
- `industry-funnel` 偏重個股篩選漏斗，從全市場逐層精選到 3 家

兩者可以互補：先用 `industry-research` 看清產業鏈格局，再用 `industry-funnel` 精選標的。

---

## 漏斗結構總覽

```
第一層：全市場掃描     30-60 家   （活躍度+漲幅+市值前 30 的並集）
        ↓ 價值投資 5 條硬指標
第二層：粗篩             ≤ 10 家   （5 條全部及格 + 護城河 ★★★ 以上）
        ↓ 精細分析
第三層：精細分析          ≤ 10 家  （每家 300-500 字結構化分析）
        ↓ 終選
第四層：四大師深度分析     3 家     （每家 800-1200 字，巴芒段李四視角）
        ↓
輸出：投資建議 + 操作訊號 + 倉位建議
```

每層"過濾掉的標的"必須留下淘汰理由，不能黑箱。

---

## 第一步：全市場掃描入口

### 1.1 活躍股票定義（三類並集）

**A 類 - 成交活躍度**：
- 近 30 天日均成交額排名行業前列（A 股/港股/美股各自取前 30）

**B 類 - 漲幅榜**：
- 近 30 天漲幅前 20
- 近 90 天漲幅前 20
- 兩者並集

**C 類 - 市值錨定**：
- 行業內市值前 30（無論漲跌）

最終掃描池 = A ∪ B ∪ C，預期 30-60 家。

### 1.2 必須搜尋的市場

| 市場 | 來源建議 |
|------|---------|
| A 股（滬深） | 同花順/東方財富行業板塊、通達信 |
| 港股 | 富途/同花順港股、HKEX 行業分類 |
| 美股 | NASDAQ/NYSE 行業 ETF 持倉、Yahoo Finance |
| 國際市場 | 日韓台歐的相關公司不能漏（特別是半導體、電子） |
| 未上市公司 | 單列"未來 IPO 候選"小節，註明最新估值與潛在 IPO 時間 |

### 1.3 輸出格式

| 公司名 | 程式碼 | 市場 | 市值 | 主業一句話 | 該行業佔比 | 入選類別(A/B/C) |
|-------|------|-----|------|----------|-----------|----------------|

**關鍵自查**：
- 行業佔比 < 30% 的"沾邊股"要謹慎，標註"非純正標的"
- 中國/亞洲市場不要因英文資料少而漏掉
- 小市值公司不要因為 AI 偏好龍頭而漏

---

## 第二步：價值投資 5 條硬指標粗篩 → ≤ 10 家

對第一步的 30-60 家公司，逐家應用 5 條硬指標。

### 2.1 5 條硬指標

| # | 指標 | 透過標準 | 放寬條件 | 資料來源 |
|---|------|---------|---------|---------|
| 1 | PE 估值 | 合理（與歷史區間、同業對比） | 高成長可放寬到 PEG < 1.5 | 財報+wind/同花順 |
| 2 | ROE | > 15% 或近 3 年趨勢改善 | 重資產行業可放寬 | 財報 |
| 3 | 經營現金流 | 為正且佔淨利潤 > 70% | — | 財報 |
| 4 | 資產負債率 | < 60% | 公用事業/電力可放寬至 70% | 財報 |
| 5 | 護城河快評 | ★★★ 以上 | — | 定性判斷 |

**護城河 5 類**：
- 品牌/定價權
- 轉換成本/使用者黏性
- 網路效應
- 規模效應
- 技術/牌照/資源壁壘

### 2.2 輸出格式

| 公司 | PE | ROE | 現金流/淨利 | 負債率 | 護城河 | 綜合 | 留/棄 | 淘汰理由 |
|------|----|----|-----------|-------|-------|------|------|---------|

**保留規則**：
- 5 條全部及格 → 直接保留
- 4 條及格 + 1 條接近 → 保留但標黃
- 不足 4 條 → 淘汰，註明理由

**目標**：保留 ≤ 10 家。如果保留過多（> 12），把護城河標準提到 ★★★★ 再篩一次。

---

## 第三步：精細分析（≤ 10 家，每家 300-500 字）

對粗篩保留的公司，逐家做結構化分析。

### 3.1 每家公司分析模板

```
## {公司名}（{程式碼}）

**一句話商業模式**：
（賣什麼、賣給誰、怎麼收錢）

**財務質量**：
- 收入增速 / 利潤增速 / 毛利率 / ROE / 現金流
- 關鍵變化（近 1-2 年最重要的財務轉折點）

**護城河深度**：
- 主要護城河型別 + 具體證據
- 護城河 5 年後是否還在的簡要判斷

**主要風險（前 3）**：
1.
2.
3.

**估值快評**：
- 當前 PE/PS/EV/EBITDA + 歷史區間位置
- 同業對比
- 一句話結論：貴 / 合理 / 便宜

**進入終選 3 家？**：是 / 否（理由）
```

### 3.2 終選 3 家的選擇標準

不是按打分排序選前 3，而是按"投資組合互補性"選：
- 至少 1 家"高確定性低彈性"（巴菲特型）
- 至少 1 家"中等確定性中等彈性"（成長型）
- 可選 1 家"高彈性高風險"（期權型）

如果某子賽道找不到 3 家足夠好的，寧可寫"終選 2 家 + 1 家觀察"，不要湊數。

## 第四步：四大師深度分析（3 家，每家 800-1200 字）

對終選 3 家執行四大師視角深度分析。

### 4.1 段永平視角：生意本質

- 一句話定義這家公司在做什麼生意
- 這是好生意嗎？為什麼？
- 它的"本分"是什麼？管理層有沒有偏離？
- 商業模式的"持續性"在哪？

### 4.2 巴菲特視角：護城河深度

- 用五類護城河打分（★1-5），列具體證據
- 10 年後護城河還在嗎？
- 現在買入的"安全邊際"在哪？

| 護城河 | 強度 | 具體證據 |
|-------|------|---------|
| 品牌/定價權 | | |
| 轉換成本 | | |
| 網路效應 | | |
| 規模效應 | | |
| 技術/牌照壁壘 | | |

### 4.3 芒格視角：風險與失敗模式

- 這家公司最可能怎麼失敗？（列出前 3 失敗路徑）
- 最壞情景下值多少錢？（極簡估值）
- 聰明人為什麼不買？（反向論證）
- 是否有道德/合規/管理層風險？

### 4.4 李錄視角：文明級趨勢定位

- 這家公司所在的賽道是"文明級正規化轉移"還是"階段性熱潮"？
- 歷史上最接近的技術革命類比？
- 10-20 年後這家公司的終局？
- 是不是贏家通吃格局？

### 4.5 綜合推薦度

```
推薦度：★★★★☆
倉位型別：核心 / 衛星 / 期權 / 觀察
建議買入區間：當前價 / 回撥 N% / 有耐心等待
建議持倉比例：佔該主題倉位 X%
關鍵監測指標：（這家公司邏輯一旦反轉的訊號是什麼）
```

## 第五步：綜合輸出

報告末尾整合：

### 5.1 終選 3 家組合表

| 公司 | 型別 | 推薦度 | 建議倉位 | 核心邏輯 | 關鍵風險 |
|------|------|-------|---------|---------|---------|
| A | 核心 | ★★★★★ | 50-60% | | |
| B | 衛星 | ★★★★☆ | 25-35% | | |
| C | 期權 | ★★★☆☆ | 5-15% | | |

### 5.2 行業級 ETF 替代

如果不想選股，列 1-3 個相關 ETF（A 股/港股/美股）。

### 5.3 整體行業位置判斷

- 行業 PE/PB 歷史分位
- 資金流向（北向、ETF 申贖、賣方覆蓋密度）
- 整體處於"早期/擴張/成熟/衰退"哪個階段

### 5.4 資訊充分度自評（必填）

| 維度 | 等級 | 說明 |
|-----|------|-----|
| 公司財務資料完整性 | A/B/C | |
| 估值資料時效性 | A/B/C | |
| 行業格局判斷 | A/B/C | |
| 管理層資訊 | A/B/C | |

A = 資料充分可信；B = 部分缺失但不影響主結論；C = 缺失較多，結論需謹慎。

### 5.5 待更新資料點

明確列出：哪些資料是估計值、哪些資料需要後續核實、哪個季度財報需要重點跟蹤。

### 5.6 資料來源清單

每個資料/結論的來源連結，分類列出（財報、研報、新聞、行業報告）。

## AI 研究偏見自覺（重要）

漏斗篩選過程中，AI 容易踩的坑：

| 偏見 | 表現 | 應對 |
|------|-----|------|
| 龍頭偏好 | 大市值公司資料多、分析篇幅長，看起來"更好" | 按硬指標和護城河打分，不按報告篇幅排序 |
| 英文偏好 | 美股資料豐富，A 股港股容易被低估 | 必須中英文都搜，A/H 公司不能漏 |
| 故事偏好 | 高漲幅 + 媒體熱度 = 看起來更好的"AI 概念股" | 區分"AI 收入佔比" vs "AI 故事佔比"，看真實業務 |
| 當下偏好 | 當前財務好的公司容易入選，可能錯過轉型期黑馬 | 第二層粗篩允許"趨勢改善"作為放寬條件 |
| 上市偏好 | 只看上市公司可能錯過賽道最好的玩家 | 必須列"未來 IPO 候選"，標註估值與時間窗 |

## 輸出要求

1. **報告位置**：`reports/{行業名}-funnel-{YYYYMMDD}.md`（行業報告放 reports/ 根目錄）
2. **語言**：中文
3. **風格**：直接、犀利、不說廢話
4. **資料**：所有資料標註來源；估計值標"估計"
5. **不預設立場**：先擺資料 → 推邏輯 → 出結論
6. **正反兩面**：每個核心判斷附反面論據
7. **每層留淘汰記錄**：被淘汰的公司也要留名字+理由

## 資料抽檢（準出流程）

報告寫入後，執行資料抽檢，透過方可釋出：

```bash
# Step 1 — 提取抽檢清單（15% 隨機抽樣）
python3 tools/report_audit.py extract \
  --report <報告檔案路徑>

# Step 2 — 對清單每項從可靠信源取數（參見 skills/financial-data.md）

# Step 3 — 輸出準出/打回判決
python3 tools/report_audit.py verdict \
  --results '<填好的JSON>' \
  --report <報告檔名>
```

**【準出】** 全部透過 → 報告可釋出；**【打回】** 有不透過 → 修正後重審。
