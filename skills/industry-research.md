# 行業投資研究：產業鏈全景掃描 + 四大師個股分析框架

對 $ARGUMENTS 行業進行系統化產業鏈投資研究。

## 研究目標

從一個投資主題/邏輯鏈出發，完成：
1. 驗證投資邏輯鏈的每一個環節
2. 繪製完整產業鏈全景圖
3. 掃描全球所有上市公司（A股/港股/美股/國際）
4. 對每個細分環節的頭部公司執行四大師框架分析
5. 輸出行業級投資組合配置建議

---

## 第一步：投資邏輯鏈構建與驗證

### 1.1 畫出邏輯鏈
用箭頭鏈路表達從"底層趨勢"到"受益標的"的因果關係，例如：
```
底層趨勢 A
    → 導致需求 B
        → 創造瓶頸/剛需 C
            → 受益產業鏈 D
```

### 1.2 逐環節驗證
對邏輯鏈的每個箭頭提出質疑並尋找證據：

| 環節 | 核心假設 | 驗證方式 | 資料來源 |
|------|---------|---------|---------|
| A→B | | 搜尋行業資料/預測 | |
| B→C | | 搜尋供需分析 | |
| C→D | | 搜尋實際案例/簽約 | |

### 1.3 尋找"已發生的驗證事件"
列出支撐該邏輯鏈的**已簽約/已落地的真實商業事件**（而非預測），例如大公司的採購協議、政策檔案、行業報告等。

---

## 第二步：產業鏈全景圖繪製

### 2.1 繪製產業鏈結構
將行業拆解為上游→中游→下游→輔助環節，例如：
```
上游：原材料/資源開採 → 材料加工/提純
中游：核心裝置製造 → 系統整合/工程建設 → 新技術研發
下游：運營/服務 → 終端客戶
輔助：檢測/認證 → 維護服務 → 金融工具（ETF/信託）
```

### 2.2 識別每個環節的"生意特徵"
對每個環節標註：

| 環節 | 商業模式 | 毛利率區間 | 競爭格局 | 壁壘型別 | 週期性 |
|------|---------|-----------|---------|---------|--------|
| | 賣資源/賣裝置/賣服務/收租 | | 壟斷/寡頭/充分競爭 | 資源/牌照/技術/規模 | 強/中/弱 |

### 2.3 標記"卡脖子環節"
識別產業鏈中供給最緊張、替代最難、利潤率最高的環節——這些往往是最佳投資標的所在。

---

## AI研究偏見自覺：行業研究的特殊陷阱

行業研究中，AI資料偏見會以獨特方式放大：

**行業級偏見**：
| 偏見型別 | 表現 | 應對 |
|---------|------|------|
| 成熟行業偏好 | 傳統行業（銀行/能源/消費）資料極多，AI分析看起來"更確定" | 確定性來自商業模式，不來自研報數量 |
| 新興行業低估 | 新行業（AI應用/合成生物等）資料少，AI分析偏保守 | 用"終局思維"而非"當前資料"判斷行業價值 |
| 龍頭偏好 | 大公司資料遠多於小公司，AI天然傾向推薦龍頭 | 小公司可能有更好的風險回報比，不要因為AI分析篇幅短就忽略 |
| 上市偏好 | 只掃描上市公司會遺漏產業鏈中的關鍵未上市玩家 | 必須搜尋未上市公司，標註"未來IPO候選" |
| 英文偏好 | AI對英文資料的處理能力更強，可能低估中國/亞洲市場玩家 | 必須同時搜尋中英文資訊源 |

**產業鏈掃描中的反偏見措施**：
1. 對每個環節，不僅列出"AI容易找到的公司"，還要主動搜尋"冷門但可能優質的標的"
2. 對資訊稀缺的小市值公司，不因分析篇幅短就降低推薦度——用核心問題（生意本質、護城河、管理層）而非報告長度來評判
3. 在最終報告中標註每家公司的"資訊充分度"（A/B/C級），讓讀者知道AI分析的可靠程度

## 第三步：全球上市公司掃描

使用 Task 工具啟動後台 Agent，全面搜尋該行業所有上市公司。

### 搜尋清單
- 美股（NYSE/NASDAQ/NYSE American）相關公司
- A股（上海/深圳）相關公司
- 港股相關公司
- 其他國際市場（日本/韓國/歐洲/澳大利亞等）
- 行業ETF
- 關鍵未上市公司（可能未來IPO）

### 對每家公司收集
- 公司名稱（中英文）
- 股票程式碼和交易所
- 市值（近似）
- 一句話描述（在產業鏈中的位置和作用）
- 是否純正標的（純核電 vs 多元化中有核電業務）
- 產業鏈所屬環節

### 輸出格式
按產業鏈環節分類，每個環節一張表，包含所有掃描到的公司。
再按投資確定性分層：
- **Tier 1**：大市值、純正標的、行業龍頭
- **Tier 2**：中市值、純正或高佔比、細分龍頭
- **Tier 3**：小市值、開發階段、高風險高彈性
- **Tier 4**：多元化公司中有相關業務的大型企業

---

## 第四步：各環節頭部公司四大師分析

對每個產業鏈環節的**Tier 1和Tier 2公司**，執行以下分析（Tier 3/4公司簡要點評即可）：

### 4.1 生意本質（段永平）
- 一句話定義這家公司在產業鏈中做什麼
- 收入結構與增速
- 毛利率/淨利率水平及趨勢
- 現金流特徵
- **追問**：這是一門好生意嗎？為什麼？

### 4.2 護城河（巴菲特）
用五類護城河評分（★1-5）：

| 護城河 | 強度 | 證據 |
|--------|------|------|
| 品牌/定價權 | | |
| 轉換成本 | | |
| 網路效應 | | |
| 規模效應 | | |
| 技術/牌照壁壘 | | |

**追問**：10年後護城河還在嗎？

### 4.3 風險（芒格）
- 這家公司最可能怎麼失敗？
- 最壞情景下值多少錢？
- 聰明人為什麼不買？

### 4.4 管理層（段永平+巴菲特）
- CEO/創始人是誰？關鍵決策記錄
- 持股比例與利益對齊
- 簡評（A/B/C級）

### 4.5 估值快照
- 當前PE/PS/EV/EBITDA
- 與同環節競爭對手對比
- 簡評：貴了/合理/便宜

### 4.6 推薦度
用★1-5標註：
- ★★★★★ = 核心倉位候選
- ★★★★☆ = 衛星倉位候選
- ★★★☆☆ = 觀察名單
- ★★☆☆☆ = 高風險期權
- ★☆☆☆☆ = 不推薦

## 第五步：行業級風險評估（芒格"檢查清單"）

### 5.1 系統性風險清單

| 風險 | 機率 | 影響 | 應對策略 |
|------|------|------|---------|
| 投資邏輯鏈的某個環節被證偽 | | | |
| 替代技術出現 | | | |
| 政策/監管黑天鵝 | | | |
| 需求週期性回撥 | | | |
| 估值泡沫破裂 | | | |

### 5.2 歷史類比
找到歷史上類似的產業鏈投資主題，分析其最終結局：
- 類比行業是什麼？
- 最終贏家是誰？（上游/中游/下游？）
- 多數投資者賺錢了還是虧錢了？
- 對當前行業的啟示是什麼？

### 5.3 偏誤自查
- 敘事偏差：故事是否太完美？
- 錨定效應：是否被近期漲幅錨定？
- 從眾效應：是否因為"所有人都在買"？

## 第六步：文明趨勢判斷（李錄框架）

- 這個行業所依託的底層趨勢，是"文明級正規化轉移"還是"階段性熱潮"？
- 歷史上最接近的技術革命類比是什麼？
- 10-20年後，這個行業的終局是什麼？
- 產業鏈中，哪個環節最可能出現"贏家通吃"？
- 哪個環節最可能被顛覆？

## 輸出要求

1. 所有分析必須有資料支撐，附資料來源
2. 使用 Markdown 表格呈現關鍵資料
3. 產業鏈全景圖用程式碼塊的文字圖表示
4. 每個環節至少分析2-3家頭部公司
5. 全球公司掃描要儘可能完整（A股/港股/美股/國際）
6. 最終將完整報告寫入 `~/[行業名]產業鏈投資研究報告.md`
7. 結論要明確，給出具體的標的、倉位和價格區間建議
8. 每個分析模組末尾有對應大師的"追問"

## 資料抽檢（準出流程）

報告寫入後，執行資料抽檢，透過方可釋出：

```bash
# Step 1 — 提取抽檢清單（15%隨機抽樣）
python3 tools/report_audit.py extract \
  --report <報告檔案路徑>

# Step 2 — 對清單每項從可靠信源取數（參見 skills/financial-data.md）

# Step 3 — 輸出準出/打回判決
python3 tools/report_audit.py verdict \
  --results '<填好的JSON>' \
  --report <報告檔名>
```

**【準出】** 全部透過 → 報告可釋出；**【打回】** 有不透過 → 修正後重審。
