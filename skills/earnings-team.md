# 財報精讀團隊：四大師並行解讀 + 公眾號釋出

對 $ARGUMENTS 進行團隊化財報精讀分析。四位大師並行解讀財報，編輯潤色成文，讀者評審把關質量，最終產出可直接釋出的公眾號文章。

**支援輸入格式**：`公司名 季度`，例如：`騰訊 2025Q4`、`PDD 2025年報`、`美團 最新`

## 設計理念

一份好的財報分析要解決兩個問題：
1. **自己能看懂未來**——需要四個不同視角的深度研究
2. **讀者能看懂價值**——需要編輯潤色和讀者視角的質量把關

本 Skill 的流程分三階段：
- **階段一·研究**：四大師並行精讀財報（段永平看生意本質、巴菲特審財務質量、芒格讀競爭變化、李錄獵風險訊號）
- **階段二·合成**：Team Lead 綜合四個視角，產出研究報告初稿
- **階段三·釋出**：編輯 Agent 改寫為公眾號文章 + 讀者評審 Agent 提出修改意見 → Team Lead 定稿

## 階段一：四大師並行研究

### 第一步：獲取一手資料

|---------|---------|--------|
| 財報原文 | 公司IR頁面、SEC EDGAR（美股）、港交所披露易（港股）、巨潮資訊網（A股） | 最高 |
| 業績電話會紀要 | Seeking Alpha、公司IR頁面、雪球 | 最高 |

| 上一期財報/電話會 | 同上 | 高（用於承諾追蹤） |

**資料可得性評級**：

| 等級 | 特徵 | 影響 |
|------|------|------|
| A級 | 獲取到完整原文 | 正常執行全部步驟 |
| B級 | 僅獲取到部分原文或第三方彙總 | 標註"非原始來源"，降低附註分析權重 |
| C級 | 僅有新聞報道和資料網站摘要 | 聚焦核心資料變化，跳過附註挖掘，標註"一手資料不足" |

將資料可得性評級告知每個 Agent，影響其分析深度。

使用 Agent 工具在**同一條訊息**中啟動4個後台 Agent。

#### Agent 1：生意本質解讀（段永平視角）

**核心問題：這份財報反映的生意本質，變好了還是變差了？**

> 段永平："投資就是買一門生意。看財報不是看數字，是看這門生意有沒有變。"

分析內容：

1. **收入結構拆解與解讀**
   - 分業務/分地區收入，哪些在加速、哪些在減速
   - 不只是列數字——每個業務板塊反映了什麼商業邏輯
   - 收入增長來自"量"還是"價"？哪種更健康？

2. **使用者/客戶價值變化**
   - DAU/MAU/付費使用者等運營指標變化
   - 使用者時長、ARPU、留存率等質量指標
   - 平台/產品對使用者的價值在增強還是減弱？

3. **護城河檢測**
   - 毛利率變化反映定價權是否穩固
   - 市場份額變化反映競爭壁壘是否有效
   - 客戶轉換成本/網路效應有沒有被削弱的訊號

4. **"好生意"標準評估**
   - 段永平三條件：差異化、定價權、可持續競爭優勢——本期變化
   - 生意在變"重"還是變"輕"？
   - 如果明天公司關門，使用者會不會非常痛苦？因為這份財報變了嗎？

5. **管理層產品直覺**
   - 管理層討論產品/使用者時，用的是具體語言還是官僚語言
   - 是否有令人印象深刻的產品洞察或令人擔憂的脫節訊號

**輸出要求**：每個子項標註 🟢改善 / 🟡持平 / 🔴惡化，給出段永平式總結點評。

---

#### Agent 2：財務質量審計（巴菲特視角）

**核心問題：這家公司賺的是真錢還是假錢？安全邊際變了嗎？**

> 巴菲特："我看每一份財報，第一件事就是翻到現金流量表。"

分析內容：

1. **核心財務資料提取與驗證**
   - 收入、毛利、經營利潤、淨利潤——GAAP和Non-GAAP都要
   - GAAP vs Non-GAAP差異：差了多少、差在哪裡、差距擴大還是縮小
   - 關鍵資料至少兩個來源交叉驗證

   ```bash
   python3 tools/financial_rigor.py cross-validate \
     --metric "revenue" --values {值1} {值2} --sources "來源1" "來源2"
   ```

2. **現金流分析（最重要）**
   - 經營現金流 vs 淨利潤比率（>100%佳，<80%警惕）
   - 自由現金流 = 經營現金流 - 資本開支
   - 資本開支構成：維護性 vs 擴張性
   - 回購和分紅金額

3. **利潤質量檢驗**
   - 應收賬款增速 vs 收入增速
   - 存貨增速 vs 收入增速
   - 經營現金流與淨利潤差距趨勢
   - 資本化支出是否突然增加
   - 非經常性收益佔比

4. **資產負債表健康度**
   - 淨現金/淨負債變化
   - 應收賬款/存貨週轉天數變化
   - 商譽及無形資產減值風險

5. **估值與安全邊際更新**

   ```bash
   python3 tools/financial_rigor.py verify-market-cap \
     --price {價格} --shares {股本} --reported {報告市值} --currency {幣種}
   python3 tools/financial_rigor.py verify-valuation \
     --price {價格} --eps {EPS} --bvps {每股淨資產}
   python3 tools/financial_rigor.py three-scenario \
     --price {價格} --eps {EPS} --shares {股本億} \
     --growth {樂觀} {中性} {悲觀} --pe {樂觀PE} {中性PE} {悲觀PE}
   ```

**輸出要求**：所有計算附工具輸出記錄，利潤質量訊號燈 🟢/🟡/🔴，巴菲特式總結點評。

---

#### Agent 3：競爭格局解讀（芒格視角）

**核心問題：這份財報揭示了競爭格局的什麼變化？**

> 芒格："我想知道我會死在哪裡，這樣我就不去那兒了。"

分析內容：

1. **從財報資料推斷競爭變化**
   - 收入增速 vs 行業增速——跑贏還是跑輸？
   - 毛利率變化反映競爭加劇/緩和
   - 營銷費用率變化——需要花更多錢獲客嗎？
   - 研發投入——主動投入還是被迫跟進？

2. **同期競爭對手對比**
   - 主要競爭對手同期關鍵指標對比（如已釋出）
   - 增速、利潤率、投入力度對比
   - 誰在贏？誰在輸？

3. **管理層對競爭的討論**
   - 電話會上如何描述競爭環境
   - 是否點名競爭對手？語氣自信還是焦慮？
   - 有沒有新的競爭威脅？

4. **行業趨勢訊號**
   - 技術變革（AI/新平台等）的影響
   - 監管變化對競爭格局的影響
   - 消費/需求端趨勢

5. **芒格式逆向思考**
   - 什麼會殺死這家公司？本期財報有沒有指向這些威脅的訊號？
   - 5年後回看，這份財報會是"轉折點"嗎？

**輸出要求**：競爭格局判斷（加強/持平/惡化），競爭對手對比表，芒格式逆向點評。

---

#### Agent 4：風險訊號獵手（李錄視角）

**核心問題：管理層在這份財報裡隱瞞了什麼？哪些訊號在閃爍？**

> 李錄："投資最重要的是避免永久性資本損失。"

分析內容：

1. **管理層語氣分析**
   - 逐段閱讀管理層討論和電話會發言，標註訊號：
   - 🟢坦誠訊號（主動承認問題）/ 🟢清晰訊號（有量化目標）
   - 🔴模糊訊號（空話）/ 🔴轉移訊號（答非所問）/ 🔴歸因外部化

2. **承諾追蹤**
   - 上一期管理層具體承諾 vs 本期實際兌現，逐條對比
   - 段永平："看管理層靠不靠譜，就看以前說的話做到了沒有。"

3. **附註與隱藏資訊**
   - 關聯交易、股權激勵稀釋、或有負債
   - 會計政策變更、分部利潤率差異
   - 客戶/供應商集中度變化

4. **電話會Q&A精選**
   - 最尖銳的3-5個分析師問題及管理層回答質量評分

5. **永久性資本損失風險**
   - 是否出現可能導致永久性損失的訊號
   - 監管/合規/訴訟風險新進展
   - 管理層是否做出了不可逆的錯誤決策

**輸出要求**：管理層可信度評分★1-5，承諾兌現率，風險訊號清單，李錄式總結點評。

## 階段二：Team Lead 合成研究報告

全部4份研究報告到齊後，Team Lead 綜合產出研究報告初稿。

**合成要點**——不是拼報告，是找交叉和矛盾：

1. **四個視角的共識點**：四位大師都同意的結論，可信度最高
2. **四個視角的矛盾點**：比如段永平說生意變好了，但芒格說競爭在惡化——這種矛盾才是最有價值的分析
3. **被忽略的角落**：四個人都沒重點提的東西，是否恰恰是最重要的？

#### 研究報告結構

```markdown
# {公司名} {期間} 財報精讀報告
**四大師並行解讀 | {日期}**

## 一、一句話結論
> 50-100字：超/符/低預期，核心變化，對投資論文的影響。

## 二、本期最重要的3個變化
聚焦真正重要的變化，不羅列資料，每個變化100字以內。

## 三、四大師評分表
| 視角 | 大師 | 核心問題 | 結論 | 評分 | vs上期 |
|------|------|---------|------|------|--------|

## 四、核心資料速覽
關鍵財務和運營指標表格（本期 vs 上期 vs 同比）

## 五、各視角深度分析
每個視角3-5條最重要發現

## 六、管理層語氣與承諾追蹤
承諾兌現表 + 語氣變化分析

## 七、四大師會怎麼做？
| 大師 | 如果持有 | 如果沒持有 | 理由 |

## 八、結論
1. 超/符/低預期？
2. 投資論文影響：強化/無影響/削弱/破裂
3. 下一個催化劑
4. 操作建議
```

## 階段三：編輯潤色 + 讀者評審

研究報告完成後，**並行**啟動兩個 Agent：

### Agent 5：編輯（公眾號文章改寫）

**定位**：把硬核研究報告改寫成公眾號讀者愛看、能看懂的文章。

**核心原則**：
- 保留所有關鍵資料和結論，不降低專業深度
- 改善表達方式，讓非專業投資者也能跟上邏輯
- 不是"科普化"，是"讓專業內容讀起來不累"

**具體任務**：

1. **標題與開頭**
   - 標題要有資訊量且吸引點選，但不做標題黨
   - 好標題示例："快手花260億賭AI，賭贏了嗎？"
   - 壞標題示例："震驚！快手財報暴雷！"
   - 開頭100字內講清楚：這份財報最重要的結論是什麼、為什麼讀者應該關心

2. **結構最佳化**
   - 研究報告是給自己看的，公眾號文章是給別人看的——調整邏輯順序
   - 把"最重要的3個變化"放在最前面（倒金字塔結構）
   - 表格保留但精簡，大段分析改為要點式
   - 每500字左右插入一個"階段性小結"，幫讀者消化

3. **表達潤色**
   - 把生硬的財務術語用類比/場景解釋："經營現金流比淨利潤低30%"→"賺了100塊但口袋裡只摸到70塊"
   - 四大師的點評語錄是文章的靈魂——確保每條都讀起來犀利、有記憶點
   - 段落不超過4行，句子不超過30字
   - 適度使用對比和反差製造閱讀節奏

4. **讀者價值檢測**
   - 每個章節自問：讀者讀完這段，能做什麼決策？如果答案是"什麼都做不了"，要麼改寫要麼刪除
   - 文章末尾需要有明確的"所以呢？"——對持有者、觀望者分別給出操作指引

5. **格式適配**
   - 微信公眾號排版友好：短段落、小標題清晰、表格簡潔
   - 加入適當的分隔線和引用格式
   - 文章長度控制在1000-3000字（太長讀者會跳出）

**輸出**：改寫後的完整公眾號文章。

---

### Agent 6：讀者評審（普通投資者視角）

**定位**：以一個"關注價值投資、有基礎財務知識、持有/關注該公司"的普通投資者身份審讀文章。

**評審維度**：

1. **可讀性（權重30%）**
   - 讀完全文需要幾分鐘？是否有想跳過的段落？
   - 哪些地方看不懂或需要反覆讀？
   - 節奏感如何？有沒有"讀累了"的感覺？

2. **資訊價值（權重30%）**
   - 讀完後，我對這家公司的理解是否加深了？
   - 有沒有"哦原來是這樣"的瞬間？
   - 和我在其他地方看到的分析相比，這篇有什麼獨到之處？
   - 哪些資訊是多餘的、刪掉也不影響理解？

3. **可信度（權重20%）**
   - 資料是否有來源？關鍵判斷是否有依據？
   - 是否呈現了正反兩面？還是隻在唱多/唱空？
   - 有沒有"這也太自信了"的判斷讓人不舒服？
   - 四大師的引用是否恰當、有力？

4. **行動指導性（權重20%）**
   - 讀完後我知道該怎麼做嗎？
   - 對"持有者"和"觀望者"的建議是否足夠具體？
   - 下一步該關注什麼？（催化劑、時間節點）

**輸出格式**：

```markdown
## 讀者評審報告

### 總體評分：X/10

### 優點（2-3條）
讀者視角下文章做得好的地方

### 必須修改（硬傷）
- 問題1：具體描述 → 建議修改方式
- 問題2：...

### 建議最佳化（錦上添花）
- 建議1：...
- 建議2：...

### 讀者最想知道但文章沒回答的問題
- 問題1
- 問題2

### 一句話總評
```

---

### Team Lead 定稿

收到編輯改寫稿和讀者評審報告後：

1. **處理讀者評審的"必須修改"項**——逐條修改
2. **選擇性採納"建議最佳化"項**——判斷是否值得
3. **補充"讀者想知道但沒回答的問題"**——如果有資料支撐就加上
4. **最終通讀**——確保修改後全文連貫、邏輯自洽

## 資料抽檢（準出流程）

對最終文章執行抽檢：

```bash
python3 tools/report_audit.py extract \
  --report reports/{公司名}/{公司名}-earnings-{期間}.md

python3 tools/report_audit.py verdict \
  --results '<填好的JSON>' \
  --report {報告檔名}
```

**【準出】** 全部透過 → 可釋出；**【打回】** 有不透過 → 修正後重審。

## 與現有 Skill 的關係

| Skill | 定位 | 何時用 |
|-------|------|--------|
| `/earnings-review` | 單Agent財報精讀 | 快速過一遍，只需一個視角 |
| **`/earnings-team`（本Skill）** | **六Agent團隊精讀 + 公眾號釋出** | **重要公司的關鍵財報，需要深度+釋出** |
| `/investment-team` | 四Agent全面公司研究 | 首次研究一家公司 |

## 關鍵原則

- **讀原文，不讀摘要**：盡一切可能獲取一手資料
- **四個視角不是四個部門**：必須相互印證和挑戰，不是各說各話
- **Team Lead 的價值在於綜合判斷**：找交集和矛盾點，不是拼報告
- **結論要明確**：不允許"總體來看基本符合預期但也有一些值得關注的點"
- **反面檢驗貫穿全程**：每個積極發現都附帶反面論據
- **編輯不是降低專業度**：是讓專業內容更易讀，不是變成科普
- **讀者評審不是走過場**：真的站在讀者角度挑毛病
- **資料準確性**：關鍵資料交叉驗證，使用 financial_rigor.py 工具驗算
