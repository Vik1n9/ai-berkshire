# 投資研究：巴菲特-芒格-段永平-李錄 四大師綜合分析框架

對 $ARGUMENTS 進行系統化投資研究分析。

## 研究框架

基於巴菲特、芒格、段永平、李錄四位投資大師的方法論，按以下七個模組順序執行研究：

### 前置步驟：AI研究偏見自覺（必須執行）

在開始研究前，先評估該公司的"AI可研究性"，識別潛在的資料偏見：

**資訊豐富度評級**：
| 等級 | 特徵 | AI研究陷阱 | 應對策略 |
|------|------|-----------|---------|
| A級（資訊充裕） | 上市多年、券商覆蓋多、媒體報道密集 | 共識過強，AI輸出趨同於市場定價，alpha有限 | 重點做反面檢驗：聰明人為什麼不買？被忽略的風險是什麼？ |
| B級（資訊適中） | 上市1-3年、覆蓋有限、部分資料需推算 | AI可能用"合理推測"填補空白，看起來完整實則虛假確定性 | 每個推算資料標註置信度，區分"有據推算"和"憑空填充" |
| C級（資訊稀缺） | 剛上市/冷門股/新興市場、幾乎無覆蓋 | AI會因資料不足而過度保守，誤判為"看不清=不好" | 用第一性原理提問（見下方），從有限資訊中提取商業本質 |

**C級公司的第一性原理研究法**：
當公開資料不足時，不要試圖拼湊出"看起來完整"的報告，而是聚焦以下底層問題：
1. 客戶是誰？為什麼付錢？有沒有替代選擇？
2. 復購靠什麼驅動？是習慣、鎖定、還是持續創造新價值？
3. 競爭對手拿100億能複製這門生意嗎？
4. 管理層做過什麼關鍵決策？這些決策反映了什麼判斷力和價值觀？

**偏見自查清單**（研究全程保持警惕）：
- [ ] 我的"確定性"感受是來自生意本質，還是來自資料數量？
- [ ] 如果把這家公司的資料量減少一半，我的結論會變嗎？
- [ ] AI輸出的分析是否與市場共識高度雷同？如果是，我的資訊優勢在哪？
- [ ] 是否存在"公開資料很少但生意本質極好"的可能性被低估了？

將資訊豐富度評級結果寫入報告開頭，並在最終結論中註明"AI研究置信度"與"實際投資確定性"的區別。

### 第一步：資料收集

> **資料來源規範**：參見 `skills/financial-data.md`。所有財務資料必須來自兩個獨立來源，誤差>1%須標記。
> - 美股：macrotrends（主）+ stockanalysis（副）
> - 港股：aastocks（主）+ macrotrends ADR（副）
> - A股：東方財富（主）+ 巨潮資訊（副）

使用 Task 工具啟動後台 Agent，從網路收集以下資料：

1. 收入結構：最近財年及近4季度分部收入、增速、毛利率
2. 財務指標：近5年收入、淨利潤、毛利率、經營利潤率、自由現金流、現金儲備
3. 競爭格局：市場份額、主要競爭對手對比
4. 商業模式與護城河：核心競爭優勢來源
5. 技術能力：核心技術棧、研發投入
6. 管理層：創始人/CEO履歷、持股比例、關鍵決策記錄
7. 行業前景：TAM（總可定址市場）、增長預測
8. 風險因素：地緣政治、監管、供應鏈等
9. 當前估值：市值、PE、PS、PEG、EV/Revenue
10. 多空雙方核心論點

#### 資料交叉驗證（必須執行，使用金融嚴謹性工具）

資料收集完成後，**必須呼叫 `tools/financial_rigor.py` 對關鍵資料進行程式化驗證**，杜絕LLM心算誤差。

**必須驗證的資料點**：
- 總股本（從交易所、Yahoo Finance、StockAnalysis 等至少2個源確認）
- 當前股價和市值（**手動計算 股價×總股本 並與報告市值對比，防止單位錯誤**）
- 最近財年收入和淨利潤（從公司年報+至少1個第三方源確認）
- 現金儲備和淨現金（現金+短期投資-總債務，注意口徑差異）
- 管理層持股比例（區分經濟權益和投票權，注意AB股結構）

**強制驗證步驟（使用Bash呼叫工具）**：

Step 1 — 市值驗算（精確十進位制，非浮點）：
```bash
python3 tools/financial_rigor.py verify-market-cap \
  --price {股價} --shares {總股本} --reported {報告市值} --currency {幣種}
```

Step 2 — 關鍵資料多源交叉驗證：
```bash
python3 tools/financial_rigor.py cross-validate \
  --field {欄位名} --values '{"來源1": 數值, "來源2": 數值}' --unit {單位}
```
對收入、淨利潤、現金儲備分別執行。

Step 3 — 估值指標精確驗算（PE/PB/ROE/FCF Yield 等）：
```bash
python3 tools/financial_rigor.py verify-valuation \
  --price {股價} --eps {EPS} --bvps {每股淨資產} --fcf-per-share {每股FCF} --dividend {每股股息}
```

**驗證規則**：
1. 每個關鍵資料點至少2個獨立來源
2. 發現來源間有差異時，優先採用公司年報/交易所資料，並註明差異原因
3. **所有涉及計算的資料必須透過工具驗算，禁止LLM心算**
4. 工具輸出結果直接嵌入報告附錄"關鍵資料交叉驗證記錄"
5. 如果工具報告 ❌ 偏差過大，必須排查原因後才能繼續分析

**常見錯誤防範**：
- 市值單位：港幣億 vs 人民幣億 vs 美元億，容易漏寫/多寫一個零
- FCF口徑：不同來源對資本支出的定義可能不同（是否含租賃、收購等）
- 債務口徑：是否包含經營租賃負債
- 持股比例：AB股公司的經濟權益 ≠ 投票權

### 第二步：生意本質分析 — 段永平"對的生意"

分析要點：
- 用一句話定義這門生意的本質
- 收入結構拆解（圖表）
- 5年盈利能力趨勢（圖表）
- 商業模式畫布：一次性銷售 vs 訂閱/復購？硬體 vs 軟體 vs 平台？
- 生態粘性/客戶鎖定強度
- 毛利率水平與同行對比，解釋為什麼高/低
- 經營槓桿分析
- **段永平式追問**：這門生意好在哪？如果只能用一句話描述，是什麼？

### 第三步：護城河評估 — 巴菲特"經濟護城河"

逐一驗證五類護城河：

| 護城河型別 | 驗證方法 |
|-----------|---------|
| 品牌/定價權 | 是否能在不損失銷量的情況下提價？ |
| 轉換成本 | 客戶遷移到競品的成本有多高？ |
| 網路效應 | 使用者越多產品越好嗎？ |
| 規模效應 | 規模帶來的成本優勢有多大？ |
| 技術/專利壁壘 | 技術領先幾年？能否被複制？ |

分析護城河趨勢：過去5年變寬還是變窄？未來5年預判。

**巴菲特式追問**：10年後這條護城河還在嗎？什麼能摧毀它？

### 第四步：逆向思考與風險清單 — 芒格"反過來想"

- 列出"這家公司可能失敗的所有路徑"（表格：路徑/機率/影響程度）
- 歷史類比：找到歷史上處於相似位置的公司，結局如何？
- 跨學科分析：用網路效應理論、技術採納曲線、競爭博弈等模型交叉驗證
- 偏誤自查：敘事偏差、錨定效應、倖存者偏差
- 收集空方核心論點

**芒格式追問**：我最可能在哪裡犯錯？聰明人為什麼會不買/做空這家公司？

### 第五步：管理層評估 — 段永平"對的人" + 巴菲特"管理層誠信"

- CEO/創始人關鍵決策覆盤（表格：時間/決策/結果/評分）
- 資本配置能力：研發回報率、併購成功率、回購時機
- 股東利益一致性：管理層持股、薪酬結構、減持記錄
- 組織能力：團隊穩定性、關鍵人才風險
- 企業文化特徵

**段永平式追問**：如果CEO退休，這家公司還能保持競爭力嗎？

### 第六步：行業與文明趨勢 — 李錄"文明演進框架"

- 判斷所在行業是否處於"文明級正規化轉移"
- 歷史技術革命類比（蒸汽機/電力/網際網路/AI）
- TAM增長曲線與天花板分析
- 公司在產業價值鏈中的位置
- 技術路線風險
- 客戶/供應商集中度分析

**李錄式追問**：站在20年後回看，這家公司是"這個時代的標準石油"還是"曇花一現的3Com"？

### 第七步：估值與安全邊際 — 巴菲特"內在價值" + 段永平"對的價格"

- 當前市場定價（關鍵估值指標表格）—— **必須透過工具驗算**

- 三情景估值 —— **必須透過工具精確計算，禁止心算**：

python3 tools/financial_rigor.py three-scenario \

- 與自身歷史估值對比
- 與同行估值對比

**段永平式追問**：如果股市明天關閉5年，你願意以這個價格持有嗎？

## 輸出要求

1. 所有分析必須有資料支撐，附資料來源
2. 使用 Markdown 表格呈現關鍵資料
3. 每個模組末尾必須有對應大師的"追問"
4. 最終將完整報告寫入 `~/[公司名]投資研究報告.md`
5. 結論要明確，不迴避給出買入/觀望/迴避的建議
6. 估值部分必須給出具體的價格區間
7. **報告開頭**必須包含"資訊豐富度評級"（A/B/C）和"AI研究侷限性宣告"
8. **報告結尾**必須區分"AI分析置信度"與"投資確定性"——前者取決於資料量，後者取決於生意本質。明確告知讀者：本報告的哪些結論基於充分資料，哪些基於有限資訊的推理
9. 如果公司屬於C級（資訊稀缺），報告末尾必須列出"需要一手驗證的問題清單"——建議讀者透過田野調查、產品體驗、供應鏈訪談等方式補充AI的盲區

## 資料抽檢（準出流程）

報告寫入檔案後，**必須**執行資料抽檢，透過後方可釋出：

**Step 1 — 提取抽檢清單（15%隨機抽樣）：**
```bash
python3 tools/report_audit.py extract \
  --report <報告檔案路徑>
```
輸出 JSON 模板，每項含 `fetched_value`（待填）。

**Step 2 — 取數核驗：**
對清單中每個資料點，按 `skills/financial-data.md` 規範從可靠信源取數
（美股：macrotrends+stockanalysis；港股：aastocks+macrotrends；A股：東方財富+巨潮資訊），
填入 `fetched_value` / `fetched_source` / `fetched_value2` / `fetched_source2`。

**Step 3 — 輸出判決：**
```bash
python3 tools/report_audit.py verdict \
  --results '<填好的JSON>' \
  --report <報告檔名>
```

- **【準出】**：所有抽檢點偏差 ≤ 1% → 報告可釋出
- **【打回】**：任意點偏差 > 1% → 修正對應資料後重新抽檢，直到準出
